기능 "$ARGUMENTS"를 검증하라.

절차:
1. `npx tsc --noEmit` 실행 — 타입 오류가 있으면 먼저 수정한다
2. qa-verifier 서브에이전트를 호출해 `docs/specs/$ARGUMENTS.md`의 "성공 기준" 체크리스트가 실제 코드에 구현되어 있는지 코드 경로 추적으로 검증하게 한다
3. 가능하면 앱을 실행해 실제 동작을 확인한다 (`npm run ios` 또는 사용자에게 실행 요청). 시뮬레이터 확인이 필요한 항목은 사용자에게 확인을 요청한다
4. qa-verifier의 보고와 앱 실행 확인 결과를 합쳐 체크리스트 형태로 보고한다: ✅ 통과 / ❌ 실패(원인과 위치) / ⏸ 수동 확인 필요
5. 실패 항목이 있으면 수정 여부를 사용자에게 묻는다

추가로 code-review 스킬이 사용 가능하면 /code-review로 변경분 리뷰를 제안한다.
