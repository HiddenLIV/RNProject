기능 "$ARGUMENTS"의 설계 문서를 작성하라.

절차:
1. `docs/specs/$ARGUMENTS.md`를 읽는다. 없으면 먼저 /spec부터 하라고 안내하고 중단한다
2. 기존 코드(`src/`)와 프로젝트 구조를 파악해 재사용할 수 있는 것을 찾는다
3. `docs/design/$ARGUMENTS.md`에 아래 구조로 작성한다:
   - **화면 구조** — 어떤 화면/컴포넌트가 필요한가, 각각 어느 파일에 두는가
   - **상태 설계** — 어떤 상태를 어디서 관리하는가 (로컬 state / context / storage)
   - **데이터 모델** — 저장할 데이터의 TypeScript 타입 정의
   - **핵심 로직** — 타이머 계산, 저장/불러오기 등 핵심 알고리즘 설명
   - **의존성** — 새로 추가할 패키지와 그 이유 (최소화할 것)
   - **구현 순서** — 어떤 순서로 만들면 단계마다 동작 확인이 가능한가
4. 스펙의 요구사항 각 항목이 설계 어디에서 충족되는지 대응시켜 확인한다
5. 설계 요약을 보여주고 사용자 승인을 받는다

주의: AGENTS.md의 코드 규칙(디렉토리 구조, 타이머는 Date.now() 기준)을 따를 것.
