description: 마무리 — reviewer 리뷰 후 변경분을 한국어 메시지로 커밋
---

기능 "$ARGUMENTS"의 변경분을 리뷰하고 커밋하라.

절차:
1. `git status`와 `git diff`로 변경분을 확인한다. git 루트는 RNProject 전체이므로 **timecheck/ 하위 파일만** 대상으로 한다
2. reviewer 서브에이전트를 호출해 변경분을 리뷰하게 한다
3. 리뷰에서 심각한 문제(버그·규칙 위반)가 나오면 커밋하지 말고 사용자에게 수정 여부를 확인한다. 스타일 수준 지적은 보고만 하고 진행한다
4. 통과하면 timecheck/ 하위 변경 파일을 스테이징하고 **한국어 커밋 메시지**로 커밋한다 (AGENTS.md 규칙). 메시지 본문에 어떤 기능의 어떤 변경인지 요약한다
5. 커밋 해시와 포함된 파일 요약을 보고한다

주의:
- push는 사용자가 명시적으로 요청할 때만 한다
- 스테이징 후 `git status`로 의도치 않은 파일(다른 프로젝트, .DS_Store 등)이 포함되지 않았는지 확인한다
- docs/specs·docs/design 문서가 코드와 어긋난 상태면 커밋 전에 문서부터 갱신한다
