# 홈 메뉴

## 목적

timecheck을 매달리기 단일 기능 앱에서 여러 운동 도구를 담는 앱으로 확장하기 위한 홈 화면. 기존 매달리기 타이머는 첫 번째 메뉴가 되고, 이후 다른 운동 기능을 메뉴로 추가한다.

## 사용자 시나리오

1. 앱을 열면 홈 화면에 운동 메뉴 리스트가 보인다.
2. "매달리기"를 선택하면 기존 타이머/기록 화면으로 들어간다.
3. 매달리기 화면에서 뒤로 가면(헤더 뒤로 버튼 또는 안드로이드 백 버튼) 홈으로 돌아온다.
4. 이후 추가될 운동 메뉴는 "준비 중"으로 표시되며 아직 선택할 수 없다.

## 요구사항

1. 앱 시작 시 홈 화면(운동 메뉴 리스트)이 먼저 표시된다.
2. 첫 메뉴는 "매달리기"이고, 선택하면 기존 매달리기 화면(타이머/기록 탭 포함)으로 이동한다. 기존 매달리기 기능(요구사항 1~19)은 그대로 동작한다.
3. 메뉴는 데이터 목록 기반으로 렌더링된다 — 코드에서 목록에 항목을 추가하기만 하면 새 메뉴가 화면에 나타난다.
4. 준비되지 않은 메뉴는 "준비 중" 표시와 함께 비활성화되어 눌러도 이동하지 않는다.
5. 매달리기 화면에서 홈으로 돌아갈 수 있다. 안드로이드 하드웨어/제스처 뒤로 가기도 동일하게 동작한다.
6. 측정(카운트다운·측정 중) 상태에서 홈으로 나가면 진행 중 측정은 취소되고 저장되지 않는다 (음성·소리도 중단).

## 제외 범위

- 다른 운동 기능의 실제 구현 — 이후 각 기능별 /spec부터 진행
- 메뉴 순서 변경·즐겨찾기 등 홈 커스터마이징
- react-navigation 등 내비게이션 라이브러리 도입 여부는 설계에서 결정

## 성공 기준

- [ ] `npx tsc --noEmit` 타입 체크 통과
- [ ] 앱 시작 시 홈 메뉴 리스트가 표시된다
- [ ] "매달리기" 선택 → 기존 타이머/기록 화면 진입, 모든 기존 기능 정상 동작
- [ ] 뒤로 가기(헤더 버튼·안드로이드 백 버튼)로 홈 복귀
- [ ] 측정 중 홈 이동 시 측정이 취소되고 음성·소리가 멈추며 기록이 저장되지 않는다
- [ ] 메뉴 목록에 항목을 추가하면 코드 수정 없이 홈에 표시된다 (준비 중 항목은 비활성)
