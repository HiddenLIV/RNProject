# 매달리기 타이머

## 목적

풀업 전 데드행(매달리기) 훈련에서 버틴 시간을 정확히 측정하고 기록으로 남겨, 사용자가 자신의 그립 지구력 향상을 추적할 수 있게 한다. 이 앱의 첫 번째이자 핵심 기능이다.

## 사용자 시나리오

1. 사용자가 앱을 열면 타이머 화면이 보인다.
2. (선택) 준비 카운트다운 시간과 벨 간격을 설정한다. 예: 카운트다운 5초, 벨 간격 30초.
3. **시작 버튼**을 누르면 설정한 시간만큼 준비 카운트다운이 시작된다. "5, 4, 3, 2, 1, 시작" 음성이 나오는 동안 봉을 잡고 매달린다.
4. 카운트다운이 끝나면 자동으로 측정이 시작되고, 경과 시간이 큰 글씨로 표시된다. 매 초마다 경과 초를 음성으로 읽어주고, 설정한 벨 간격(기본 10초)의 배수에서는 음성 대신 벨 소리가 재생되어 화면을 보지 않아도 경과 시간을 알 수 있다.
5. 더 이상 버틸 수 없으면 봉에서 내려와 **정지 버튼**을 누른다.
6. 측정 결과가 화면에 표시되고, 기록이 자동으로 저장된다.
7. 세트를 반복하려면 결과 화면의 **다시 시작 버튼**으로 같은 설정 그대로 즉시 새 측정을 시작한다.
8. 기록 목록에서 과거 측정 기록(날짜/시각, 버틴 시간)을 확인할 수 있고, 최고 기록을 볼 수 있다.

## 요구사항

### 타이머 측정

1. 시작 버튼을 누르면 준비 카운트다운이 표시된 후 측정이 자동 시작된다.
2. 준비 카운트다운 시간은 사용자가 3~30초 범위에서 초 단위로 설정할 수 있다 (기본값 5초). −/+ 버튼 외에 값을 탭해 숫자를 직접 입력할 수도 있다 (범위 밖 입력은 3~30으로 보정). 설정값은 저장되어 다음 측정에서도 유지된다.
3. 카운트다운 중 취소 버튼으로 측정을 취소할 수 있다 (기록 저장 안 됨).
4. 측정 중 경과 시간이 0.01초 단위로 화면에 크게 표시된다.
5. 경과 시간은 시작 시각(`Date.now()`) 기준으로 계산한다 — 앱이 백그라운드로 갔다 돌아와도 시간이 정확해야 한다.
6. 정지 버튼을 누르면 측정이 종료되고 결과(버틴 시간)가 표시된다.
7. 결과 화면에서 다시 시작 버튼으로 같은 설정(카운트다운·벨 간격) 그대로 즉시 새 측정을 시작할 수 있다 (카운트다운부터 다시 진행).

### 음성·소리 피드백

8. 준비 카운트다운은 남은 초를 음성으로 읽어준다 (예: 5초 설정 시 "5, 4, 3, 2, 1, 시작").
9. 측정 중 매 초마다 경과 초를 음성으로 읽어준다 (예: "일", "이", "삼" …).
10. 벨 간격의 배수 초에서는 숫자 음성 대신 구별되는 벨 소리를 재생한다. 벨 간격은 10~120초 범위에서 10초 단위로 설정할 수 있고(기본 10초 — 예: 30초로 설정하면 30, 60, 90초에 벨), 설정값은 저장되어 다음 측정에서도 유지된다.
11. 음성·소리는 측정이 정지·취소되면 즉시 멈춘다.
12. 측정·카운트다운 중에는 화면이 자동 잠금되지 않는다 (잠금 시 JS 타이머가 멈춰 음성·벨이 끊기는 것 방지).
13. 측정(카운트다운 포함)이 시작되면 기기에서 재생 중이던 다른 앱의 음악·영상이 일시정지되고, 측정 중에는 재개되지 않는다 (측정 종료 후 재개 여부는 해당 앱의 동작을 따른다).

### 동기부여 문구

14. 측정 중 화면에 운동 동기부여 문구(포기하지 말라는 명언)가 3초마다 랜덤으로 교체 표시된다 (직전 문구 연속 반복 없음, 음성 없이 화면 표시만).

### 기록 저장·조회

15. 측정이 정상 종료되면 기록(측정 일시, 버틴 시간)이 로컬(AsyncStorage)에 자동 저장된다. 단, 1초 미만 기록은 오조작으로 보고 저장하지 않는다 (결과 화면에 미저장 안내 표시).
16. 기록 목록 화면에서 과거 기록을 일자별 섹션으로 묶어 최신순으로 볼 수 있다 (섹션 헤더는 날짜·요일, 각 행은 시각·버틴 시간).
17. 최고 기록(가장 오래 버틴 시간)이 목록 상단 카드로 항상 먼저 보이고, 목록 안에서도 해당 기록이 강조된다.
18. 개별 기록을 삭제할 수 있다 (잘못 측정한 경우).
19. 앱을 종료했다 다시 열어도 기록이 유지된다.

## 제외 범위

- 통계/그래프 (주간·월간 추이 등) — 이후 기능
- 목표 시간 설정·도달 알림 — 초기 스펙에 있었으나 사용자 요청으로 제거 (필요 시 이후 재도입)
- 음성·소리 켜기/끄기 설정 — 이후 기능 (이번에는 항상 켜짐)
- 수동 잠금 상태에서의 음성 재생 — 사용자가 전원 버튼으로 직접 잠그면 음성·벨은 멈춘다 (경과 시간은 Date.now() 기준이라 정확히 유지됨). 자동 잠금만 keep-awake로 방지
- 세트 관리 (여러 세트 묶음, 휴식 타이머) — 이후 기능
- 클라우드 동기화, 계정 — 로컬 저장만
- Android/iOS 위젯, Apple Watch 연동

## 성공 기준

- [ ] `npx tsc --noEmit` 타입 체크 통과
- [ ] 시작 → 준비 카운트다운 → 측정 시작이 자동으로 이어진다
- [ ] 카운트다운 시간을 3~30초 범위에서 변경할 수 있고, 변경값이 앱 재시작 후에도 유지된다
- [ ] 카운트다운 값을 탭해 숫자를 직접 입력할 수 있고, 범위 밖 입력(예: 99)은 3~30으로 보정된다
- [ ] 카운트다운이 남은 초 음성("5, 4, 3, 2, 1, 시작")으로 안내된다
- [ ] 카운트다운 중 취소하면 기록이 저장되지 않는다
- [ ] 측정 중 경과 시간이 0.01초 단위로 갱신되어 표시된다
- [ ] 측정 중 매 초 경과 초가 음성으로 나온다
- [ ] 벨 간격 배수 초에서는 숫자 음성 대신 벨 소리가 재생된다
- [ ] 벨 간격을 10~120초 범위에서 10초 단위로 변경할 수 있고, 변경값이 앱 재시작 후에도 유지된다
- [ ] 정지·취소 시 음성·소리가 즉시 멈춘다
- [ ] 측정 시작 후 첫 번째 벨(예: 10초)이 정상 재생된다 (안드로이드 최초 실행 포함)
- [ ] 측정 중 화면이 자동 잠금되지 않고, 측정 종료 후에는 정상 잠금된다
- [ ] 측정 중 앱을 백그라운드로 보냈다 복귀해도 경과 시간이 실제 시간과 일치한다
- [ ] 다른 앱에서 음악 재생 중 측정을 시작하면 음악이 일시정지되고 측정 중 재개되지 않는다
- [ ] 측정 중 동기부여 문구가 3초마다 바뀌어 표시된다
- [ ] 정지 시 결과가 표시되고 기록이 자동 저장된다
- [ ] 결과 화면의 다시 시작 버튼으로 같은 설정의 새 측정이 카운트다운부터 시작되고, 직전 결과는 기록에 남아 있다
- [ ] 기록 목록이 일자별 섹션으로 묶여 최신순으로 표시되고, 최고 기록이 목록 상단 카드 + 목록 내 강조로 표시된다
- [ ] 개별 기록 삭제가 동작한다
- [ ] 앱 재시작 후에도 기록이 유지된다
