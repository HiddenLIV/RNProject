# 홈 메뉴 — 설계

스펙: `docs/specs/홈 메뉴.md` (요구사항 1~6)

## 화면 구조

화면 전환이 홈 ↔ 매달리기 2단계뿐이므로 내비게이션 라이브러리 없이 App.tsx 로컬 state로 전환한다 (기존 타이머/기록 탭과 같은 방식).

| 파일 | 역할 |
|---|---|
| `App.tsx` | 화면 루트: `screen: 'home' \| 'hang'` state 전환, 안드로이드 백 버튼 처리, 오디오 모드 설정 |
| `src/lib/menu.ts` | 운동 메뉴 데이터 목록 (`ExerciseMenu[]`) — 여기에 항목 추가만 하면 홈에 표시 |
| `src/screens/HomeScreen.tsx` | 메뉴 리스트 렌더링. available 항목 탭 → 해당 화면 진입, 미구현 항목은 "준비 중" 비활성 |
| `src/screens/HangScreen.tsx` | 매달리기 컨테이너: 헤더(뒤로 버튼+제목) + 타이머/기록 탭 전환 (기존 App.tsx의 탭 UI 이동) |

TimerScreen·RecordsScreen 등 기존 매달리기 화면·로직은 수정 없이 HangScreen 아래로 이동만 한다.

## 데이터 모델

```ts
// src/lib/menu.ts
export type ExerciseMenu = {
  id: string;          // 'hang' 등 — App.tsx screen 값과 연결
  title: string;       // '매달리기'
  description: string; // 한 줄 소개
  available: boolean;  // false면 "준비 중" 비활성
};

export const EXERCISE_MENUS: ExerciseMenu[] = [
  { id: 'hang', title: '매달리기', description: '데드행 버틴 시간 측정·기록', available: true },
];
```

## 핵심 로직

1. **화면 전환** — App.tsx `screen` state. HomeScreen에 `onSelect(id)` 콜백, HangScreen에 `onBack` 콜백 전달.
2. **안드로이드 백 버튼** (요구사항 5) — App.tsx에서 `BackHandler.addEventListener('hardwareBackPress')`: screen이 'hang'이면 홈으로 전환하고 true 반환, 'home'이면 false(기본 동작=앱 종료).
3. **측정 중 이탈 정리** (요구사항 6) — 화면 전환으로 TimerScreen이 언마운트되면 기존 효과 cleanup이 interval·keep-awake·벨·무음 루프를 해제한다. 여기에 언마운트 시 `stopFeedback()`(TTS 중단) cleanup을 추가해 진행 중이던 음성도 즉시 멈춘다. 저장은 원래 정지 버튼 경로에서만 일어나므로 자동으로 미저장.

## 의존성

추가 없음. `BackHandler`는 react-native 내장.

## 구현 순서

1. `menu.ts` + HomeScreen (리스트 UI)
2. HangScreen (기존 App.tsx 탭 UI 이동 + 헤더)
3. App.tsx 재구성 (screen 전환 + BackHandler)
4. TimerScreen 언마운트 시 stopFeedback cleanup 추가
5. 타입 체크 + 실기기 확인

## 스펙 요구사항 대응표

| 요구사항 | 설계 반영 위치 |
|---|---|
| 1. 앱 시작 시 홈 표시 | App.tsx 초기 state 'home' |
| 2. 매달리기 메뉴 → 기존 화면 | HangScreen이 기존 탭 UI를 그대로 포함 |
| 3. 데이터 목록 기반 메뉴 | menu.ts EXERCISE_MENUS + HomeScreen FlatList |
| 4. 준비 중 비활성 | available=false → disabled + "준비 중" 배지 |
| 5. 뒤로 가기 | HangScreen 헤더 뒤로 버튼 + App.tsx BackHandler |
| 6. 측정 중 이탈 시 취소·음성 중단 | 효과 cleanup + TimerScreen 언마운트 stopFeedback |
