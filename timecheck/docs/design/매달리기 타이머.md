# 매달리기 타이머 — 설계

스펙: `docs/specs/매달리기 타이머.md` (요구사항 1~15)

## 화면 구조

내비게이션 라이브러리 없이 App.tsx에서 로컬 state로 두 화면을 전환한다 (하단 탭 형태). 화면이 2개뿐이라 react-navigation은 과하다.

| 파일 | 역할 |
|---|---|
| `App.tsx` | 탭 전환(타이머/기록), 앱 시작 시 오디오 모드 설정 |
| `src/screens/TimerScreen.tsx` | 측정 화면. phase에 따라 대기(설정+시작 버튼) / 카운트다운 / 측정 중 / 결과 UI |
| `src/screens/RecordsScreen.tsx` | 기록 목록(최신순), 최고 기록 강조, 개별 삭제 |
| `src/components/TimeDisplay.tsx` | 경과 시간 큰 글씨 표시 (`MM:SS.cc` 포맷, 0.01초 단위) |
| `src/components/SecondsStepper.tsx` | 초 단위 값 조절 UI (− / 값 / +). 카운트다운·벨 간격 설정에 공용 |
| `src/components/RecordItem.tsx` | 기록 한 줄 (일시, 버틴 시간, 삭제 버튼, 최고 기록 배지) |
| `src/lib/useHangTimer.ts` | 타이머 상태 머신 훅 (시간 로직만, 소리 없음) |
| `src/lib/feedback.ts` | 음성(TTS) 재생/중지 — 벨 소리는 `useAudioPlayer`가 훅이라 TimerScreen에서 직접 재생 |
| `src/lib/storage.ts` | AsyncStorage 기록·설정 읽기/쓰기 |
| `src/lib/types.ts` | 데이터 타입 정의 |

## 상태 설계

- **타이머 상태** — `useHangTimer` 훅이 관리하는 상태 머신:
  ```
  idle → (start) → countdown → (0 도달) → running → (stop) → finished → (reset) → idle
                       └─ (cancel) → idle
  ```
  `phase`, `elapsedMs`, `countdownRemainingSec`를 노출하고, 초 경계 이벤트를 콜백(`onCountdownSecond`, `onMeasureSecond`, `onMeasureStart`)으로 알린다. 음성·벨 소리는 TimerScreen이 이 콜백에 연결한다(TTS는 `feedback.ts`, 벨은 `useAudioPlayer`) — 시간 로직과 사이드이펙트를 분리해 검증을 쉽게 한다.
- **설정** (`Settings`) — TimerScreen 로컬 state. 마운트 시 storage에서 로드, 변경 즉시 저장. (요구사항 2, 9)
- **기록** (`HangRecord[]`) — RecordsScreen 로컬 state, 마운트 시 로드. 탭 전환이 화면을 unmount/mount 하므로 타이머 화면에서 저장한 기록이 탭 이동 시 자연히 반영된다. 저장은 TimerScreen이 정지 시 `storage.addRecord()` 호출.
- **최고 기록** — 별도 저장하지 않고 records에서 파생 계산 (`Math.max(durationMs)`).

## 데이터 모델

```ts
// src/lib/types.ts
export type HangRecord = {
  id: string;          // `${Date.now()}-${랜덤}` — 삭제 시 식별자
  measuredAt: string;  // ISO 8601 측정 일시
  durationMs: number;  // 버틴 시간 (밀리초)
};

export type Settings = {
  countdownSeconds: number;        // 3~30, 기본 5
  bellIntervalSeconds: number;     // 10~120, 10초 단위, 기본 10
};

export const DEFAULT_SETTINGS: Settings = {
  countdownSeconds: 5,
  bellIntervalSeconds: 10,
};
```

AsyncStorage 키 (JSON 직렬화):
- `timecheck:records:v1` — `HangRecord[]`, 최신 기록을 배열 앞에 추가 (목록이 곧 최신순)
- `timecheck:settings:v1` — `Settings`

## 핵심 로직

1. **경과 시간 계산** (요구사항 5) — 시작 시 `startedAt = Date.now()` 저장. 33ms `setInterval`(약 30fps)은 리렌더 트리거일 뿐이고, 매 tick마다 `elapsed = Date.now() - startedAt`으로 재계산한다. 카운트다운도 동일하게 `endsAt` 기준. 백그라운드 전환·프레임 드랍에도 오차 없음 (AGENTS.md 규칙).
2. **초 경계 감지** — `lastAnnouncedSecond` ref를 두고 tick마다 `s = Math.floor(elapsed / 1000)` 비교. `s`가 증가했으면 **최신 초 하나만** 이벤트 발생 — 백그라운드에 있다 복귀해 여러 초가 건너뛰어져도 밀린 음성을 몰아서 읽지 않는다.
3. **음성·소리 분기** (요구사항 7~9) — `onMeasureSecond(s)`에서 `s % settings.bellIntervalSeconds === 0`이면 벨 소리(`await player.seekTo(0); player.play()`), 아니면 `Speech.speak(String(s), { language: 'ko-KR' })`. 벨 간격은 SecondsStepper(step=10)로 10~120초 설정. 카운트다운은 남은 초를 speak, 0 도달 시 "시작" speak 후 measuring 진입.
4. **정지·취소** (요구사항 10) — `Speech.stop()` + `player.pause()` + interval 해제. 취소(카운트다운 중)는 저장 없이 idle 복귀, 정지(측정 중)는 `addRecord` 후 finished. 단, 1초 미만이면 오조작으로 보고 저장하지 않고 결과 화면에 미저장 안내를 표시한다. 벨 소리는 재생 직전에 `await seekTo(0)` 후 `play()`하므로 정지 시 되감기가 필요 없다.
5. **iOS 무음 스위치 대응** — 앱 시작 시 `setAudioModeAsync({ playsInSilentMode: true })` 1회 호출 (App.tsx). 헬스장에서 무음 모드여도 음성이 나와야 하므로 필요.
6. **저장** — `storage.ts`는 read-modify-write (`getRecords` → 수정 → `AsyncStorage.setItem`). 파싱 실패 시 빈 배열 폴백.

## 의존성

`npx expo install`로 설치 (SDK 57 호환 버전 자동 선택):

| 패키지 | 이유 |
|---|---|
| `@react-native-async-storage/async-storage` | 기록·설정 로컬 저장 (AGENTS.md 지정) |
| `expo-speech` | 초 읽기 TTS — SDK 57 문서 확인 완료 (`Speech.speak`/`stop`) |
| `expo-audio` | 10초 배수 효과음 — SDK 57 문서 확인 완료 (`useAudioPlayer` 훅). `expo-av`는 구세대라 사용 안 함 |

- 벨 소리 에셋: `assets/sounds/bell.wav` — `scripts/make-bell-sound.js`로 생성해 번들 (외부 다운로드·저작권 이슈 없음). 앱 아이콘도 `scripts/make-icons.js`로 재생성 가능.

## 구현 순서

각 단계 끝에서 시뮬레이터로 동작 확인이 가능하도록 나눈다.

1. **타입 + 저장소** — `types.ts`, `storage.ts` 작성, 타입 체크 통과
2. **타이머 코어** — `useHangTimer.ts` + TimerScreen 기본 UI. 시작→카운트다운→측정→정지 흐름과 0.01초 표시를 소리 없이 확인
3. **설정** — SecondsStepper로 카운트다운·벨 간격 설정, 저장·복원 확인
4. **음성·벨 소리** — 패키지 설치, `bell.wav` 생성, `feedback.ts` 작성 후 콜백 연결
5. **기록** — 정지 시 저장, RecordsScreen(목록·최고 강조·삭제), App.tsx 탭 전환
6. **마무리** — `npx tsc --noEmit`, 스펙 성공 기준 자체 점검 → `/qa`

## 스펙 요구사항 대응표

| 요구사항 | 설계 반영 위치 |
|---|---|
| 1. 카운트다운 후 자동 시작 | useHangTimer 상태 머신 (countdown→running 자동 전이) |
| 2. 카운트다운 3~30초 설정·유지 | SecondsStepper + Settings.countdownSeconds + storage |
| 3. 카운트다운 중 취소 | useHangTimer `cancel()` — 저장 없이 idle 복귀 |
| 4. 0.01초 단위 표시 | 33ms tick + TimeDisplay |
| 5. Date.now() 기준 계산 | 핵심 로직 1 |
| 6. 정지 시 결과 표시 | phase=finished UI (TimerScreen) |
| 7. 카운트다운 음성 | onCountdownSecond → Speech.speak |
| 8. 매 초 음성 | onMeasureSecond → Speech.speak (핵심 로직 3) |
| 9. 벨 간격 배수 효과음 (간격 설정 가능) | onMeasureSecond의 `s % bellIntervalSeconds` 분기 + expo-audio + SecondsStepper(step=10) |
| 10. 정지·취소 시 소리 중단 | 핵심 로직 5 (`Speech.stop` + player 정지) |
| 11. 정지 시 자동 저장 (1초 미만 제외) | storage.addRecord (핵심 로직 4) |
| 12. 최신순 목록 | records 배열 앞 추가 + RecordsScreen |
| 13. 최고 기록 강조 | 파생 계산 + RecordItem 배지 |
| 14. 개별 삭제 | storage.removeRecord(id) + RecordItem 삭제 버튼 |
| 15. 재시작 후 유지 | AsyncStorage 영속화 |
